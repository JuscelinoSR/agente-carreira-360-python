# Política de segurança

## Versões suportadas

Somente a versão mais recente da branch `main` recebe correções de segurança.

## Como relatar uma vulnerabilidade

Não publique credenciais, dados pessoais ou detalhes exploráveis em issues públicas.

Use o recurso **Report a vulnerability** na aba **Security** do repositório. Caso ele ainda não esteja habilitado, abra uma issue sem detalhes sensíveis solicitando um canal privado de contato.

Inclua no relato privado:

- descrição do problema;
- impacto esperado;
- passos mínimos para reprodução;
- sugestão de correção, quando houver.

## Escopo de segurança

- vazamento de dados profissionais ou credenciais;
- escrita ou leitura de arquivos fora das pastas esperadas;
- publicação externa sem confirmação humana;
- uso inseguro de tokens de GitHub, LinkedIn ou outras APIs.
